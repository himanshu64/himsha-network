Quick Setup

Start the full stack locally with Docker Compose before running any tutorial.

1
Clone the repo and build 
git clone https://github.com/your-org/himsha-network.git
cd himsha-network
cargo build --workspace
2
Start Bitcoin + Electrs + Ord + HIMSHA Node 
cd bitcoin-infra
cp .env.example .env
# Edit .env: set BITCOIN_RPC_USER and BITCOIN_RPC_PASS
mkdir -p data/{bitcoin,electrs,ord,him}
docker compose up -d

# Wait for all services to be healthy (~2 minutes on regtest)
docker compose ps
3
Fund the regtest wallet (regtest only) 
bash scripts/init-regtest.sh
# Output: Wallet balance: 50.00000000 BTC
4
Verify the HIMSHA node is ready 
curl -s -X POST http://localhost:9100 \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"himsha_isNodeReady","params":[]}'

# Expected: {"result":true}

Create Your First Account

All state on HIMSHA Network lives in accounts. The System Program creates them.

import { HimshaConnection, HimshaPublicKey, HimshaTransaction } from "@himsha-network/sdk";
import { SystemProgram } from "@himsha-network/sdk";

const conn   = new HimshaConnection("http://localhost:9100");
const payer  = HimshaPublicKey.fromSeed("my-wallet");    // derive from keypair in production
const newAcc = HimshaPublicKey.fromSeed("my-data-account");
const owner  = HimshaPublicKey.fromSeed("my-program");

const tx = HimshaTransaction.create([payer], [
  SystemProgram.createAccount(
    payer,
    newAcc,
    1_000_000n,   // lamports (rent)
    256n,         // data space in bytes
    owner,        // which program owns this account
  )
]);

const txId = await conn.sendTransaction(tx);
console.log("Account created, txId:", txId);

// Verify
const info = await conn.getAccountInfo(newAcc);
console.log("Account:", info);
from himsha_sdk import HimshaConnection, HimshaPublicKey, HimshaTransaction
from himsha_sdk.programs import system

async with HimshaConnection("http://localhost:9100") as conn:
    payer   = HimshaPublicKey.from_seed("my-wallet")
    new_acc = HimshaPublicKey.from_seed("my-data-account")
    owner   = HimshaPublicKey.from_seed("my-program")

    tx = HimshaTransaction.create(
        signers=[payer],
        instructions=[
            system.create_account(payer, new_acc, 1_000_000, 256, owner)
        ]
    )

    tx_id = await conn.send_transaction(tx)
    print(f"Account created: {tx_id}")

    info = await conn.get_account_info(new_acc)
    print(f"Lamports: {info.lamports}")
import 'package:himsha_sdk/himsha_sdk.dart';

final conn   = HimshaConnection('http://localhost:9100');
final payer  = HimshaPublicKey.fromSeed('my-wallet');
final newAcc = HimshaPublicKey.fromSeed('my-data-account');
final owner  = HimshaPublicKey.fromSeed('my-program');

final tx = HimshaTransaction.create([payer], [
  SystemProgram.createAccount(payer, newAcc, BigInt.from(1000000), BigInt.from(256), owner),
]);

final txId = await conn.sendTransaction(tx);
print('Account created: $txId');

conn.close();

Create a Token

A token has two accounts: a Mint (controls supply and decimals) and a Token Account (holds balances for a specific wallet).

Flow:
  1. Create the Mint account (System Program)
  2. Initialize the Mint (Token Program)
  3. Create a Token Account for the holder
  4. Initialize the Token Account
// TypeScript
import { SystemProgram, TokenProgram, PROGRAM_IDS } from "@himsha-network/sdk";

const mint         = HimshaPublicKey.fromSeed("mytoken-mint");
const mintAuth     = HimshaPublicKey.fromSeed("mytoken-authority");
const tokenAccount = HimshaPublicKey.fromSeed("alice-mytoken-account");
const alice        = HimshaPublicKey.fromSeed("alice");

const tx = HimshaTransaction.create([payer, mintAuth], [
  // Step 1: allocate space for the mint (Mint state = ~82 bytes)
  SystemProgram.createAccount(payer, mint, 2_000_000n, 82n, PROGRAM_IDS.token),

  // Step 2: initialize mint (6 decimals, mintAuth can mint, no freeze)
  TokenProgram.initializeMint(mint, mintAuth, 6),

  // Step 3: create token account for Alice
  SystemProgram.createAccount(payer, tokenAccount, 2_000_000n, 165n, PROGRAM_IDS.token),

  // Step 4: initialize token account
  TokenProgram.initializeAccount(tokenAccount, mint, alice),
]);

await conn.sendTransaction(tx);
console.log("Token created!");

Mint & Transfer Tokens

// Mint 1,000 tokens (6 decimals = 1_000_000_000 raw units)
const mintTx = HimshaTransaction.create([mintAuth], [
  TokenProgram.mintTo(mint, tokenAccount, mintAuth, 1_000_000_000n)
]);
await conn.sendTransaction(mintTx);

// Transfer 100 tokens from Alice to Bob
const bobAccount = HimshaPublicKey.fromSeed("bob-mytoken-account");
const bob        = HimshaPublicKey.fromSeed("bob");

const transferTx = HimshaTransaction.create([alice], [
  TokenProgram.transfer(tokenAccount, bobAccount, alice, 100_000_000n)
]);
await conn.sendTransaction(transferTx);

console.log("Transferred 100 tokens from Alice to Bob");

Burn Tokens & Close Account

// Burn 50 tokens from Alice's account
const burnTx = HimshaTransaction.create([alice], [
  TokenProgram.burn(tokenAccount, mint, alice, 50_000_000n)
]);
await conn.sendTransaction(burnTx);

// Close Alice's empty token account, reclaim lamports
// Account must have 0 token balance before closing
const closeTx = HimshaTransaction.create([alice], [
  TokenProgram.closeAccount(tokenAccount, payer, alice)  // lamports → payer
]);
await conn.sendTransaction(closeTx);
console.log("Account closed, lamports reclaimed");

Create a Liquidity Pool

The Swap Program uses the constant-product formula x · y = k. A pool consists of two token reserves and an LP mint for liquidity shares.

Accounts needed: pool state account, tokenA mint, tokenB mint, reserveA token account, reserveB token account, LP mint. 
import { SwapProgram, PROGRAM_IDS } from "@himsha-network/sdk";

const pool     = HimshaPublicKey.fromSeed("btc-usdc-pool");
const mintA    = HimshaPublicKey.fromSeed("wBTC-mint");   // token A
const mintB    = HimshaPublicKey.fromSeed("USDC-mint");   // token B
const reserveA = HimshaPublicKey.fromSeed("pool-reserveA");
const reserveB = HimshaPublicKey.fromSeed("pool-reserveB");
const lpMint   = HimshaPublicKey.fromSeed("pool-lp-mint");

const tx = HimshaTransaction.create([payer], [
  // Create pool state account
  SystemProgram.createAccount(payer, pool, 2_000_000n, 256n, PROGRAM_IDS.swap),

  // Initialize pool: 0.3% fee (3/1000)
  SwapProgram.initialize(
    pool, mintA, mintB, reserveA, reserveB, lpMint, payer,
    3n,     // feeNumerator
    1000n,  // feeDenominator → 0.3%
  )
]);

await conn.sendTransaction(tx);
console.log("Liquidity pool created with 0.3% fee");

Add Liquidity

Deposit both tokens to receive LP tokens. On first deposit, any ratio is accepted. Subsequent deposits must match the existing ratio (or excess is returned). 

const userA  = HimshaPublicKey.fromSeed("alice-tokenA");
const userB  = HimshaPublicKey.fromSeed("alice-tokenB");
const userLp = HimshaPublicKey.fromSeed("alice-lp");

const depositTx = HimshaTransaction.create([alice], [
  SwapProgram.deposit(
    pool, userA, userB, reserveA, reserveB, userLp, alice,
    1_000_000n,    // maxA: up to 1 BTC worth of tokenA
    60_000_000n,   // maxB: up to 60,000 USDC
    1n,            // minLP: accept any LP tokens
  )
]);

await conn.sendTransaction(depositTx);
console.log("Liquidity added. You received LP tokens.");

Swap Tokens

Exchange tokenA for tokenB (or vice versa). Set minAmountOut to protect against slippage. 

// Swap 0.01 BTC worth of tokenA for at least 590 USDC (1% slippage tolerance)
const swapTx = HimshaTransaction.create([alice], [
  SwapProgram.swap(
    pool,
    userA,     // source (Alice's tokenA account)
    userB,     // destination (Alice's tokenB account)
    reserveA,  // pool reserve for tokenA (input side)
    reserveB,  // pool reserve for tokenB (output side)
    alice,     // signer
    10_000n,   // amountIn: 0.01 BTC
    590_000n,  // minAmountOut: reject if <590 USDC
  )
]);

await conn.sendTransaction(swapTx);
console.log("Swapped tokenA → tokenB");
Slippage formula: The AMM outputs amountOut = reserveOut * amountIn / (reserveIn + amountIn). Set minAmountOut to at least 99% of expected output to limit slippage.

Remove Liquidity

// Burn 500 LP tokens to receive proportional tokenA + tokenB
const withdrawTx = HimshaTransaction.create([alice], [
  SwapProgram.withdraw(
    pool, userA, userB, reserveA, reserveB, userLp, alice,
    500n,     // lpAmount: burn 500 LP tokens
    4_500n,   // minA: expect at least this much tokenA
    270_000n, // minB: expect at least this much tokenB
  )
]);

await conn.sendTransaction(withdrawTx);
console.log("Liquidity removed. LP tokens burned.");

Initialize a Lending Collection

A Collection is a lending market for one NFT collection (e.g. "FrogNFTs"). Lenders post bids; borrowers pledge inscriptions to receive BTC. 

import { LendingProgram } from "@himsha-network/sdk";

const collection = HimshaPublicKey.fromSeed("frog-nfts-lending-market");

const tx = HimshaTransaction.create([payer], [
  SystemProgram.createAccount(payer, collection, 2_000_000n, 4096n, PROGRAM_IDS.lending),
  LendingProgram.initCollection(collection, payer, "FrogNFTs")
]);

await conn.sendTransaction(tx);
console.log("FrogNFTs lending market created!");

Place a Bid (Lender)

A lender commits a Bitcoin UTXO as loan funds and specifies the loan value, duration, and repayment address. Multiple lenders compete; borrowers pick the best offer. 

// Alice (lender) offers 0.001 BTC for 30 days
const bidTxid = new Uint8Array(32).fill(0xaa); // real txid bytes in production

const tx = HimshaTransaction.create([alice], [
  LendingProgram.placeBid(
    collection,
    alice,                          // lender signs
    bidTxid,                        // UTXO containing the loan funds
    0,                              // vout
    100_000n,                       // loanValue: 100,000 sats (0.001 BTC)
    BigInt(30 * 24 * 3600),         // loanPeriod: 30 days in seconds
    "tb1q_alice_ordinals_addr",     // where NFT goes on default
    "tb1q_alice_payments_addr",     // where repayment goes
  )
]);

await conn.sendTransaction(tx);
console.log("Bid placed! Waiting for a borrower...");

Accept a Bid (Borrower)

The borrower pledges a Bitcoin inscription as collateral. The node automatically selects the highest bid, transfers loan funds to the borrower, and locks the inscription. 

// Bob (borrower) pledges his Frog inscription and receives 0.001 BTC
const inscriptionTxid = new Uint8Array(32).fill(0xbb); // inscription UTXO

const tx = HimshaTransaction.create([bob], [
  LendingProgram.acceptBid(
    collection,
    bob,                              // borrower signs
    "frog42i0",                      // inscription ID
    inscriptionTxid,                  // inscription UTXO txid
    0,                                // inscription vout
    "tb1q_bob_ordinals_addr",        // where inscription returns on repay
    "tb1q_bob_payments_addr",        // where Bob receives loan funds
  )
]);

await conn.sendTransaction(tx);
console.log("Loan accepted! Bob received 0.001 BTC. Inscription is locked.");

Repay a Loan

Bob repays within the 30-day window. The inscription returns to Bob; Alice receives her 0.001 BTC back. 

// Bob repays within the loan period
const repaymentTxid = new Uint8Array(32).fill(0xcc); // UTXO with >= 100,000 sats

const tx = HimshaTransaction.create([bob], [
  LendingProgram.repayLoan(
    collection,
    bob,
    "frog42i0",     // inscription to reclaim
    repaymentTxid,  // UTXO paying back the loan
    0,
  )
]);

await conn.sendTransaction(tx);
console.log("Loan repaid! Inscription returned to Bob.");

Claim Default (Lender)

If Bob doesn't repay after 30 days, Alice can claim the inscription. The ZK proof verifies the timestamp is past the deadline. 

// Alice claims after 30+ days of no repayment
const tx = HimshaTransaction.create([alice], [
  LendingProgram.claimDefault(
    collection,
    alice,       // lender signs
    "frog42i0", // defaulted inscription
  )
]);

await conn.sendTransaction(tx);
console.log("Default claimed! Frog inscription transferred to Alice.");
⚠️ Timing: The ZK prover reads Message.timestamp. If the timestamp is before the deadline, the proof fails with ProgramError::LoanNotExpired. The node rejects invalid transactions before they reach the prover.

Create NFT Metadata

import { NftMetadataProgram, PROGRAM_IDS } from "@himsha-network/sdk";

const metadataAccount = HimshaPublicKey.fromSeed("frog42-metadata");
const mint            = HimshaPublicKey.fromSeed("frog42-mint");
const updateAuth      = HimshaPublicKey.fromSeed("collection-authority");

const tx = HimshaTransaction.create([payer, updateAuth], [
  SystemProgram.createAccount(payer, metadataAccount, 2_000_000n, 512n, PROGRAM_IDS.nftMetadata),

  NftMetadataProgram.create(
    metadataAccount,
    mint,
    updateAuth,
    "Frog #42",                           // name (max 32 chars)
    "FROG",                                // symbol (max 10 chars)
    "https://frogs.xyz/api/metadata/42",  // URI (max 200 chars)
    500,                                    // 5% royalty
    [{ address: updateAuth, share: 100, verified: false }],
  )
]);

await conn.sendTransaction(tx);
console.log("NFT metadata created!");

Update & Freeze Metadata

// Update the URI (while still mutable)
const updateTx = HimshaTransaction.create([updateAuth], [
  NftMetadataProgram.update(
    metadataAccount, updateAuth,
    { uri: "https://frogs.xyz/api/metadata/42?v=2" }
  )
]);
await conn.sendTransaction(updateTx);

// Make immutable — no more updates possible
const freezeTx = HimshaTransaction.create([updateAuth], [
  NftMetadataProgram.makeImmutable(metadataAccount, updateAuth)
]);
await conn.sendTransaction(freezeTx);
console.log("Metadata frozen. Cannot be updated.");

Run Rust Tests

Each program crate has a #[cfg(test)] module covering all instructions, edge cases, and error paths.

# Run all Rust tests
cargo test --workspace

# Run tests for a specific program
cargo test -p himsha-token-program
cargo test -p himsha-swap-program
cargo test -p himsha-lending-program
cargo test -p himsha-runtime

# Run tests with output (shows println! in tests)
cargo test --workspace -- --nocapture

# Run a specific test by name
cargo test test_swap_constant_product -- --nocapture
