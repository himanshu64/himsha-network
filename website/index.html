Why HIMSHA

Architecture

How HIMSHA works

Four layers, each with a single clear responsibility.

STEP 01

Client submits a transaction

A client builds a RuntimeTransaction — a list of signed instructions targeting one or more HIMSHA programs. Signed with standard Bitcoin Schnorr keys. No new wallet software needed. 

// TypeScript SDK
const tx = HimshaTransaction.create(
  [signer],
  [LendingProgram.initCollection(
    collectionAccount, payer, "FrogNFTs"
  )]
);
await conn.sendTransaction(tx);
STEP 02

Node executes in a ZK prover

The HIMSHA node passes the transaction to the RISC Zero zkVM. The program runs inside a sandboxed RISC-V environment. When it finishes, the prover emits a cryptographic receipt proving the execution was correct. 

// himsha-vm executor (Rust)
let receipt = default_prover()
  .prove_elf(env, &prog.elf)?;

receipt.verify(prog.image_id)?;
// ↑ Anyone can verify this
STEP 03

State anchored to a Bitcoin UTXO

Updated account state is committed to a Bitcoin UTXO. Even if every HIMSHA validator disappears, the state can be reconstructed from the Bitcoin chain. Your assets stay on L1. 

// Account anchored to Bitcoin
AccountInfo {
  key:  Pubkey,
  data: Vec<u8>,        // program state
  utxo: Some(UtxoMeta {
    txid: [u8; 32],     // Bitcoin txid
    vout: 0,
  }),
}
STEP 04

Bitcoin transaction broadcast

If the program produces a Bitcoin transaction (e.g. transferring an inscription), the node signs and broadcasts it. The UTXO moves on L1. No bridge. No wrapped tokens. Real Bitcoin. 

// Bitcoin indexer (Rust)
let txid = indexer
  .broadcast(&signed_tx_hex)?;

// inscription moved on L1 ✓
println!("confirmed: {txid}");

SDKs

Build in your language

Three SDKs, one API surface. Ship a frontend, mobile app, or server without learning a new paradigm. 

// npm install @himsha-network/sdk
import { HimshaConnection, LendingProgram, HimshaPublicKey } from "@himsha-network/sdk";

const conn = new HimshaConnection("http://localhost:9100");
const collection = HimshaPublicKey.fromSeed("my-nft-collection");

// Initialize a lending market
const tx = HimshaTransaction.create([payer], [
  LendingProgram.initCollection(collection, payer.publicKey, "FrogNFTs"),
]);
tx.addSignature(await wallet.sign(tx.messageHash()));

const txId = await conn.sendTransaction(tx);
console.log(`Collection created: ${txId}`);
# pip install himsha-sdk
from himsha_sdk import HimshaConnection, HimshaPublicKey, HimshaTransaction
from himsha_sdk.programs import lending

async with HimshaConnection("http://localhost:9100") as conn:
    collection = HimshaPublicKey.from_seed("my-nft-collection")

    tx = HimshaTransaction.create(
        signers=[payer],
        instructions=[
            lending.init_collection(collection, payer, "FrogNFTs")
        ]
    )
    tx.add_signature(wallet.sign(tx.message_hash()))

    tx_id = await conn.send_transaction(tx)
    print(f"Collection created: {tx_id}")

Quick Start

Up and running in 3 steps

1

Clone and build

git clone https://github.com/your-org/himsha-network.git
cd himsha-network
cargo build --workspace
2

Start the stack with Docker

cd bitcoin-infra
cp .env.example .env          # edit credentials
mkdir -p data/{bitcoin,electrs,ord,him}
docker compose up -d
3

Check node status

curl -X POST http://localhost:9100 \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"himsha_isNodeReady","params":[]}'

# {"jsonrpc":"2.0","result":true,"id":1}

FAQ

Frequently asked questions

What is HIMSHA Network?

HIMSHA (Hashable Instruction Machine) is a fully open-source, experimental Bitcoin programmability layer. Every state transition is proven correct by a RISC Zero zero-knowledge receipt rather than by validator majority vote. It is an educational proof of concept and is not production-grade.

How is HIMSHA different from other Bitcoin L2s?

Correctness comes from a ZK proof (math) instead of validator consensus or a fraud-proof window. HIMSHA requires no bridges, makes privacy possible, and does not depend on validator honesty for correctness.

Do I need a bridge or wrapped Bitcoin to use HIMSHA?

No. Account state is anchored to Bitcoin UTXOs and real Bitcoin transactions are broadcast on Layer 1. There are no bridges and no wrapped tokens.

What programs are built into HIMSHA?

Eight programs ship in the box: System, Token, ATA, Swap (AMM), Lending against Ordinal inscriptions, NFT Metadata, Runes, and Money Market.

Which languages have an SDK?

TypeScript, Python, and Dart — all sharing one API surface, so you can build a frontend, mobile app, or server.

Is HIMSHA production-ready?

No. HIMSHA is an educational proof of concept. It is not audited and must not be used with real Bitcoin mainnet funds.
